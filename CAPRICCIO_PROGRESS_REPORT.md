## 3. Принятые решения

| Решение | Обоснование |
|---------|------------|
| Vite + React + Tailwind | Быстрый старт, совместимость с Capacitor и PWA |
| Supabase | PostgreSQL + Auth + Storage + Realtime в одном месте, бесплатный старт |
| Zustand | Лёгкий стейт-менеджер, без бойлерплейта Redux |
| PWA + Capacitor | Один кодовый репозиторий для Web + iOS + Android |
| Vercel | Простой деплой из репозитория, бесплатный tier |
| Qwen 3.5 для UI | Экономия бюджета на простых задачах |
| Claude Sonnet 4.6 для логики и интеграций | Точность на сложных задачах |
| Раздельные роли (user / admin / service_role) | Безопасность на уровне БД через RLS |
| Партнёрская БД отдельными таблицами | Изолированность данных, чистая логика начислений |

## 4. Текущий статус

### Работает
- Проект запускается (`npm run dev`) без ошибок
- Роутинг работает — все маршруты зарегистрированы
- Supabase подключён, ключи внесены в `.env`
- База данных развёрнута — 14 таблиц, RLS, триггеры, индексы
- Storage buckets созданы
- Авторизация через Supabase Auth готова к использованию

### Не реализовано
- UI компоненты (Header, Footer, все страницы)
- Каталог, карточка товара, корзина
- Личный кабинет, партнёрский кабинет
- Интеграции (Billz, Kommo, Telegram)
- Оплата
- Админка
- Блог

### Проблем нет
Проект в рабочем состоянии, готов к UI-разработке.
